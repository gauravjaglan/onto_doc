Description
Ideas can exist independently or in connection to other ideas, and are what we use to make connections. Notice that 'abstract_idea' here can also include composite ideas, i.e. sets of connections or one single connection used as a node in an argument. By doing so we can support, for example, Dung's abstract argumentation framework.

Class: cohere.owl:connection   back to top

  • cohere.owl:abstract_idea
    • cohere.owl:connection

NOTE cohere.owl:connection is a leaf node.

Description
A connection of two abstract_ideas through a relation. Since a connection is itself an abstract_ideas, the connection can also link two connections, or one connection with a node (= a simple idea) . in other words, a connection can be used as a node in a new connection. At the moment, we only provide support for representing binary connections only (two nodes only). Moreover, when a connection is instantiated, it is also possible to define the node_types of the two ideas which are connected. The node_types are also called the 'roles' or 'contexts' of the ideas.

NOTE cohere.owl:connection_node_role is a leaf node.

Description
It is the contextual information associated with the idea in a connection: we can also refer to it as the 'role' of the idea in that connection. For example, it can be an 'answer', a 'conclusion', an 'idea', an 'experiment' etc. Cohere suggests some predefined roles, but users can also create new roles for their ideas.

NOTE cohere.owl:set is a leaf node.

Description
This class serves as the basic mean to create abstractions within Cohere. By creating a set it is possible to have any arbitrary aggregation of connections, nodes or sets themselves into a new abstract_idea. In other words, the resulting set is itself an idea that can be connected into a new argument.
